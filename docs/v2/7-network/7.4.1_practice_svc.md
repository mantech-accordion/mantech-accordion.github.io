## 실습 문제


<details>
<summary>예제1 Yaml</summary>
  
{% highlight yaml %}

---
apiVersion: apps/v1
kind: Deployment
metadata:
  name: demo-success-nginx
  namespace: edu-namespace0 # 본인의 네임스페이스 지정
  labels:
    app: demo-success-nginx
spec:
  replicas: 1
  selector:
    matchLabels:
      app: demo-success-nginx
  template:
    metadata:
      labels:
        app: demo-success-nginx
    spec:
      containers:
      - name: nginx
        image: base.member1.accordions.co.kr:5000/nginx:1.31.0-alpine
        ports:
        - containerPort: 80

---
apiVersion: v1
kind: Service
metadata:
   name: demo-success-nginx
spec:
  selector:
    app: demo-success-nginx
  ports:
  - name: tcp
    port: 80
    protocol: TCP
    targetPort: 80
  type: NodePort

{% endhighlight %}
   
</details>

**1. 예제1 yaml을 사용하여 Deployment 및 Service를 생성하세요.**
```
- 네임스페이스 : 본인의 네임스페이스로 설정하세요.
```

**2. 생성한 서비스를 NodePort로 접근하여 확인하세요.**

HINT
```
- URL=> http://<IP>:<NodePort>
```

**3. 예제2 yaml을 사용하여 Deployment 및 Service를 생성하세요.**

**4. 생성한 서비스를 NodePort로 접근하여 확인하세요.**

HINT
```
URL=> http://<IP>:<NodePort>
```

**5. 접근이 되지 않는 서비스를 찾고, 그 이유를 확인하세요.**

**6. 생성한 deployment 및 Service를 모두 삭제하세요.**
