# 고급 배포 전략

블루/그린 및 카나리 배포 전략을 설정하여 무중단 배포 기능을 제공합니다.<br>
고급 배포 전략은 하나의 워크로드만 지정이 가능하며, haproxy Ingress Provider만 지원합니다.<br>
고급 배포 전략 구성 시, 해당 카탈로그의 마지막 이력은 성공 상태여야 하며,<br>
워크로드(deployment, statefulset)와 연결된 Service와 Ingress 각 1개 이상 존재해야합니다.<br>
Ingress를 통한 연결이 정상적이지 않다면 무중단 배포 초기화 시, 구성이 실패할 수 있습니다.

---

# 리소스 관리

**고급 배포 전략**을 통하여 배포를 진행할 경우 아래와 같이 중복 리소스 사용이 발생 됩니다.<br>
<br>
- 원본 리소스 1개<br>
- 운영 리소스 1개 (원본 리소스를 기준으로 복제한 리소스)<br>
- 과거 버전 리소스 N개<br>
<br>
과거 버전을 몇개 유지하느냐에 따라 자원 소비율이 올라가게 됩니다.<br>
과거 버전 유지 개수를 적절하게 설정을 해주는 것이 중요합니다.<br>
과거 버전 유지를 1개로 할 경우 최대 상정한 리소스의 3배가 할당 됨으로, 주의가 필요합니다.<br>
원본 리소스 같은 경우 운영 리소스 배포 이후 삭제를 해도 무방하나<br>
실수로 원본 리소스 삭제 후 고급 배포 전략을 삭제하여 운영 리소스가 삭제 될 경우<br>
**서비스 중단**이 발생하게 됩니다. (동작중인 Pod가 없어진 것이 사유)

주의 사항

고급 배포 전략을 처음 구성하거나 제거 시, 해당 운영 인그레스 동작이 일시적으로 끊길 수 있으므로 생성, 삭제 시 주의한다.<br>
고급 배포 전략 구성 시, 기본적으로 기존 사용하던 리소스 자원을 최소 두 배 이상 사용되므로 충분한 자원 사용량이 확보되어야 한다.
