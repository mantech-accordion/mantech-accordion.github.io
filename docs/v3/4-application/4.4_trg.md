## 트리거

트리거는 카탈로그/파이프라인을 대상으로 하며, 이벤트 발생을 기준으로 2가지로 분류됩니다.


| 종류        |  설명  |
|:------------|:-------|
| 예약 액션 트리거 | 주기적으로 등록된 이벤트를 감지, 대상이 되는 카탈로그/파이프라인을 실행 |
| 빌드 후 액션 트리거 | 카탈로그/파이프라인이 배포 또는 빌드된 이후 동작을 수행 |

![catalog_trigger_list.png](/assets/images/v3/application/trigger/catalog_trigger_list.png)

---

## 트리거 등록

카탈로그 혹은 파이프라인에서 `트리거` 탭을 선택한 뒤 `트리거 생성` 버튼을 클릭하고, `예약 액션 트리거` 또는 `빌드 후 액션 트리거`를 선택합니다.

![catalog_trigger_create_observers.png](/assets/images/v3/application/trigger/catalog_trigger_create_observers.png)
![catalog_trigger_create_afterhooks.png](/assets/images/v3/application/trigger/catalog_trigger_create_afterhooks.png)

클릭하면 `이름`, `타입`, `옵션`을 입력할 수 있는 화면이 나타납니다.
- 이름은 해당 트리거에 대한 이름을 설정할 수 있습니다.
- 타입은 트리거의 타입을 선택할 수 있습니다.
- 옵션은 선택한 트리거에서 필요한 옵션의 키와 밸류를 입력합니다.

---

### 예약 액션 트리거

예약 액션 트리거는 등록된 스케줄에 따라 주기적으로 동작하며 **이벤트가 감지** 되면 빌드(배포)를 수행합니다.

![catalog_trigger_create_schedule.png](/assets/images/v3/application/trigger/catalog_trigger_create_schedule.png)

예약 액션 트리거 등록시에는 `스케줄` 셀렉트박스를 클릭하면 기본으로 설정되어 있는 스케줄을 설정할 수 있습니다.<br>
Cron 방식으로 등록하며, **5분 이상** 으로 등록하길 권장합니다.<br><br>

직접 스케줄 설정 작성시 아래와 같이 크론 스케줄 포맷에 의해 작성할수도 있습니다.

[스케줄 설정의 예시]
```
1 9 1 11 *
 = 매년 11월 1일 오전 9시 1분에 동작
 ```

 아래는 사용 가능한 예약 액션 트리거의 종류입니다.

 |  트리거 종류  |  설명  |
|:------------|:-------|
| 버전관리 시스템의 저장소 변경 감지 | 빌드의 메타데이터를 기반으로 원격저장소의 변경을 감지 |
| 컨테이너 이미지 변경 감지 | 빌드의 메타데이터를 기반으로 컨테이너 이미지 변경을 감지 또는 빌드의 `spec.env.EXTERNALIMAGE` 정보를 기반으로 이미지의 변경을 감지 |
| http 요청에 대한 응답 감지 | http 요청을 통해서 응답 데이터를 감지 |
| 정기적으로 트리거 실행 | 등록된 스케줄 주기 마다 빌드(배포)를 수행 |

---

### 빌드 후 액션 트리거

사용 가능한 빌드 후 액션 트리거의 종류입니다.

| 종류        |  설명  |
|:------------|:-------|
| 카탈로그/파이프라인 빌드(배포) 트리거 | 같은 클러스터의 다른 카탈로그/파이프라인을 빌드(배포) |
| 웹훅 전달 트리거 | 설정된 URL로 카탈로그/파이프라인 정보와 빌드 정보를 HTTP POST 요청으로 전달한다. |

## 트리거 목록

해당 카탈로그 또는 파이프라인에 등록된 트리거들의 목록을 확인할 수 있습니다.

![catalog_trigger_list.png](/assets/images/v3/application/trigger/catalog_trigger_list.png)

`실행` 토글 버튼으로 트리거 동작 여부를 설정할 수 있습니다. <br>
`변경 상태`는 해당 트리거의 상태 정보 이며 `예약 액션 트리거` 의 경우 `성공`과 `실패`가 있으며, <br>
`빌드 후 액션 트리거`의 경우 `성공`, `실패`, `스킵` 이 존재합니다.

| 종류        |  설명  |
|:------------|:-------|
| 성공 | 트리거가 감지하여 작업을 수행한 경우 |
| 실패 | 트리거가 작업도중 실패한 경우 |
| 스킵 | 감지는 했으나 조건이 일치하지 않은 경우, 이미 빌드 중인 경우 |

`최근 트리거` 는 해당 트리거의 최근 동작 시간이며, 예약 액션 트리거 의 경우 성공 또는 실패 시간만 나타납니다.
`수행 주기` 는 ``예약 액션 트리거 에서만 볼수 있고 해당 트리거의 스케줄을 나타냅니다.

## 트리거 상세

원하는 트리거를 클릭 하면 상세 정보를 확인할 수 있습니다.

![catalog_trigger_result.png](/assets/images/v3/application/trigger/catalog_trigger_result.png)

`예약 액션 트리거` 의 경우, 상단에서 `실행 여부`, `상태 메세지` 를 확인할 수 있고 <br>
하단 표에서 `조건 감지`, `최근 체크시간`, `키`, `밸류` 를 확인할 수 있습니다. <br><br>

트리거의 `조건 감지` 종류는 아래와 같습니다.
`조건 감지` 는 `예약 액션 트리거` 의 `트리거의 상태 정보` 와 동일 하며 `미감지`와 `스킵` 이 추가 됩니다.

| 종류        |  설명  |
|:------------|:-------|
| 미감지 | `예약 액션 트리거` 에서만 사용되며 트리거가 감지되지 않은 경우 |

최근 체크 시간 은 해당 트리거의 스케줄에 따라 주기적으로 감지를 시도한 시간입니다.<br><br>

`빌드 후 액션 트리거` 의, 경우 상단에서 `실행 여부`, `상태 메세지` 를 확인할 수 있고
하단 표에서 키, 밸류 를 확인할 수 있습니다.

## 트리거 수정

`트리거` 우측의 `수정` 버튼을 클릭 하면 수정을 할 수 있는 상태가 됩니다.<br>
수정이 완료가 되면 `수정하기` 버튼을 클릭하여 저장할 수 있습니다.

![catalog_trigger_edit.png](/assets/images/v3/application/trigger/catalog_trigger_edit.png)

---

## 트리거 삭제

삭제를 원하는 트리거의 우측 `삭제` 버튼을 클릭하면 모달이 나타납니다.<br>
모달에서 트리거 이름을 입력 후 `삭제`합니다.

![catalog_trigger_delete.png](/assets/images/v3/application/trigger/catalog_trigger_delete.png)
