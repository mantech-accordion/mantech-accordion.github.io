## 클러스터 대시보드
클러스터 대시보드는 클러스터의 쿠버네티스 리소스 및 CPU/메모리/네트워크와 같은 시스템 리소스 사용 현황 정보를 한눈에 제공합니다.

### 제어 영역
화면 우측 상단에 다음과 같은 제어 기능이 제공됩니다.

|항목|설명|
|:--|:--|
|조회 기간|시스템 리소스 차트(CPU, 메모리, 네트워크 사용량)의 조회 기간을 설정한다. 기본값은 최근 5분 이다. 선택 가능한 값은 다음과 같다: 최근 5분 , 최근 10분 , 최근 30분 , 최근 1시간 , 최근 3시간 , 최근 6시간 , 최근 12시간 , 최근 24시간 , 최근 2일 , 최근 7일 , 직접 설정하기|
|새로고침|대시보드 데이터를 수동으로 새로고침한다.|
|자동 새로고침|대시보드 데이터를 주기적으로 자동 새로고침하는 간격을 설정한다. 기본값은 끄기 이다. 선택 가능한 값은 다음과 같다: 끄기, 5초, 10초, 30초, 1분, 5분, 15분, 30분, 1시간|
|대시보드 수정|대시보드 레이아웃을 편집하는 모드로 전환한다. 위젯을 수정하거나 삭제할 수 있다. |

### 클러스터 정보 카드
화면 상단에 클러스터의 기본 정보를 요약하는 정보 카드가 표시됩니다.

|항목|설명|
|:--|:--|
|클러스터 이름|클러스터 이름이 굵은 글씨로 표시된다 (예: host-cluster-181).|
|Role|클러스터 역할 (예: Host)|
|Provider|쿠버네티스 제공자 (예: Server)|
|K8S|쿠버네티스 버전 (예: v1.34.2) |
|Accordion|아코디언 버전 (예: 3.0.0)|
|사용자|현재 클러스터에 등록된 사용자 수|
|그룹|현재 클러스터에 등록된 그룹 수|

### 시스템 리소스 차트
클러스터의 시스템 리소스 사용 현황을 실시간 차트로 제공합니다. 조회 기간 설정에 따라 차트의 시간 범위가 변경됩니다.

**CPU**

CPU 사용률을 도넛 차트와 시계열 라인 차트로 표시합니다.

|항목|설명|
|:--|:--|
|도넛 차트|현재 CPU 사용률을 백분율로 시각화한다. 중앙에 CPU 와 전체 코어 수가 표시된다.|
|라인 차트|단위: %. 선택한 조회 기간 동안의 CPU 사용률 추이를 시계열 그래프로 표시한다.|


**메모리**

메모리 사용률을 도넛 차트와 시계열 라인 차트로 표시합니다.

|항목|설명|
|:--|:--|
|도넛 차트|현재 메모리 사용률을 백분율로 시각화한다. 중앙에 메모리 와 전체 메모리 용량(GB)이 표시된다.|
|라인 차트|선택한 조회 기간 동안의 메모리 사용률 추이를 시계열 그래프로 표시한다.|


**네트워크 사용량**

네트워크 수신량과 송신량을 시계열 라인 차트로 표시합니다.

|항목|설명|
|:--|:--|
|수신량|파란색 선으로 네트워크 수신량 추이를 표시한다.|
|송신량|빨간색 선으로 네트워크 송신량 추이를 표시한다.|

차트 하단의 범례에서 수신량 과 송신량을 확인할 수 있습니다.

### 이벤트
화면 우측에 클러스터에서 발생한 쿠버네티스 이벤트 목록이 실시간으로 표시됩니다. 이벤트 목록 상단의 접기/펼치기 아이콘으로 이벤트 영역을 접거나 펼 수 있습니다.

**이벤트 필터**

우측 드롭다운에서 이벤트 유형을 필터링할 수 있습니다.
- All: 모든 이벤트 표시 (기본값)
- Normal: 정상 이벤트만 표시
- Warning: 경고 이벤트만 표시

**이벤트 목록**

각 이벤트 항목은 다음과 같은 정보를 포함합니다.

|항목|설명|
|:--|:--|
|시간|이벤트 발생 시각 (예: 오후 10:48)|
|상태 표시|이벤트 유형에 따라 색상이 다르게 표시된다. 정상 이벤트는 초록색, 경고 이벤트는 노란색 등으로 구분된다.|
|메시지|이벤트 요약 메시지가 한 줄로 표시된다.|

**이벤트 상세**

이벤트 항목을 클릭하면 상세 정보가 펼쳐집니다.

|항목|설명|
|:--|:--|
|네임스페이스|이벤트가 발생한 네임스페이스|
|리소스명|관련 리소스의 종류 배지와 이름 (예: PersistentVolumeClaim 배지, chatbot-nvdia-g8j8f)|
|원인|이벤트 발생 원인 (예: WaitForFirstConsume)|
|재시작 횟수|관련 컨테이너의 재시작 횟수|
|메시지|이벤트 상세 메시지|

### 쿠버네티스 리소스 정보
클러스터에 배포된 주요 쿠버네티스 리소스의 현황을 요약 카드로 제공합니다. 각 카드는 빨간색 상단 테두리로 구분되며, 정상 상태 수와 비정상 상태 수가 함께 표시됩니다. 비정상 상태 수는 빨간색으로 강조 표시됩니다.

**노드**

|항목|설명|
|:--|:--|
|Ready|정상 상태인 노드 수|
|NotReady|비정상 상태인 노드 수 (빨간색으로 표시)|

**네임스페이스**

|항목|설명|
|:--|:--|
|Active|활성 상태인 네임스페이스 수|
|Terminating|삭제 중인 네임스페이스 수|

**파드**

|항목|설명|
|:--|:--|
|Running|실행 중인 파드 수|
|Completed|완료된 파드 수|
|NotRunning|실행되지 않은 파드 수 (빨간색으로 표시) |

**퍼시스턴트볼륨**

|항목|설명|
|:--|:--|
|Bound|바인딩된 퍼시스턴트볼륨 수|
|NotBound|바인딩되지 않은 퍼시스턴트볼륨 수 (빨간색으로 표시)|

**퍼시스턴트볼륨클레임**

|항목|설명|
|:--|:--|
|Bound|바인딩된 퍼시스턴트볼륨클레임 수|
|NotBound|바인딩되지 않은 퍼시스턴트볼륨클레임 수 (빨간색으로 표시)|

**인그레스**

|항목|설명|
|:--|:--|
|Count|클러스터에 배포된 인그레스 총 수|

### Top 차트
클러스터의 리소스 사용 현황을 상위 항목 기준으로 막대 차트로 제공합니다.

**CPU 사용량 Top 7**

노드별 CPU 사용량을 높은 순서대로 최대 7개까지 가로 막대 차트로 표시합니다. 단위는 core 이며, 각 항목 옆에 사용량 수치가 표시됩니다.

**메모리 사용량 Top 7**

노드별 메모리 사용량을 높은 순서대로 최대 7개까지 가로 막대 차트로 표시합니다. 단위는 GB 이며, 각 항목 옆에 사용량 수치가 표시됩니다.

**네임스페이스별 파드 개수**

네임스페이스별 파드 배포 현황을 높은 순서대로 가로 막대 차트로 표시합니다. 단위는 개수 이며, 각 항목 옆에 파드 수가 표시됩니다.

**전체 파드 상태**

클러스터 내 전체 파드의 상태별 분포를 가로 막대 차트로 표시합니다. 각 상태는 색상이 지정된 원형 아이콘과 함께 표시됩니다.

|항목|설명|
|:--|:--|
|Running|실행 중인 파드 수|
|Completed|완료된 파드 수|
|Pending|대기 중인 파드 수|
|Terminating|종료 중인 파드 수|
|Failed|실패한 파드 수|
|Unknown|상태를 알 수 없는 파드 수|
|etc|기타 상태의 파드 수|
