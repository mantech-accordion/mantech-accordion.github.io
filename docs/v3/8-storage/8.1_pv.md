## 퍼시스턴트볼륨(PersistentVolume)

퍼시스턴트볼륨 (PV)은 관리자가 프로비저닝하거나 스토리지 클래스를 사용하여 동적으로 프로비저닝한 클러스터의 스토리지입니다.
퍼시스턴트 볼륨의 유형 퍼시스턴트볼륨 유형은 플러그인으로 구현됩니다. 쿠버네티스는 현재 다음의 플러그인을 지원합니다.

## 퍼시스턴트볼륨 생성

`+생성`을 클릭 후 내용을 입력하여 서비스를 생성할 수 있습니다. 서비스는 FORM/YAML로 입력할 수 있습니다.

일반 탭에서는 이름과 네임스페이스, 라벨, 어노테이션을 설정할 수 있습니다.

![pv-003-0.png](/assets/images/v3/storage/pv/pv_create_normal.png)

설정 탭에서는 소스, 엑세스모드, 볼륨모드를 선택할 수 있습니다.

![pv-003-1.png](/assets/images/v3/storage/pv/pv_create_detail.png)

**퍼시스턴트볼륨(PV) 설정 항목**

| 소스 유형 | 하위 설정 항목 | 설명 |
|:---|:---|:---|
| **호스트경로**<br>(HostPath) | **경로 타입 (type)** | `DirectoryOrCreate`: 경로가 없으면 생성<br>`Directory`: 기존 디렉터리만 허용<br>`FileOrCreate`: 파일이 없으면 생성<br>`File`: 기존 파일만 허용<br>`기본값("")`: 타입 검사 없음 |
| | **경로 (path)** | 노드 내 실제 디렉터리 또는 파일 경로 |
| **로컬** | **경로** | 노드의 실제 마운트 경로 |
| | **노드어피니티** | 매칭할 노드 라벨의 키, 연산자(In, Exists 등), 값을 설정 |
| **NFS** | **주소 (server)** | NFS 서버 IP 또는 도메인 |
| | **경로 (path)** | NFS 서버 내 공유 디렉터리 경로 |
| | **읽기전용** | `true` / `false` |

수정하려는 퍼시스턴트볼륨을 선택하고 우측 상단의 수정 버튼을 클릭합니다.

![pv-004.png](/assets/images/v3/storage/pv/pv_edit.png)

## 퍼시스턴트볼륨 삭제

삭제하려는 퍼시스턴트볼륨를 선택하고 우측의 삭제 버튼을 선택합니다. 모달에서 퍼시스턴트볼륨를 입력하여 삭제합니다.
