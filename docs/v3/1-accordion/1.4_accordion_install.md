## 개요

`accordion-installer-ui`는 아코디언 인스톨러를 웹 기반에서 실행할 수 있는 기능을 제공하는 콘솔입니다.
아코디언 인스톨러, 아코디언 업그레이드, 엣지 인스톨러에서 사용하는 설정 파일을 웹에서 편집하고, 설치 스크립트 실행, 로그와 보고서를 함께 관리할 수 있습니다.


## 지원 범위

### 설정 파일

UI에서 설정 가능한 범위는 아래와 같습니다.

| 구분 | 지원 파일 |
| --- | --- |
| 아코디언 인스톨러 | `노드 설정`, `호스트 클러스터 설정`, `멤버 클러스터 설정`, `Kubeconfig 호스트 클러스터 조인`, `Kubeconfig 멤버 클러스터 조인` |
| 아코디언 업그레이드 | `노드 설정`, `호스트 클러스터 업그레이드 설정`, `멤버 클러스터 업그레이드 설정` |
| 엣지 인스톨러 | `노드 설정`, `엣지 클러스터 설정` |

## 설치 전 준비

### 사전 준비

- Podman 설치
- 아코디언 인스톨러, 업그레이드, 엣지 인스톨러 스크립트 압축 해제(필요한 것만)
- 기본 경로 예시
  - `/opt/accordion-installer`
  - `/opt/accordion-upgrade`
  - `/opt/accordion-edge-installer`

### 컨테이너 기동

기동 스크립트: `scripts/run.sh`

- 볼륨 마운트 대상 선택: `accordion`, `upgrade`, `edge`, `all`
- 관리자 계정은 애플리케이션 시작 시 자동 생성되며, `run.sh`로 기동하면 관리자 비밀번호를 자동 생성해서 출력합니다.

```bash
$ cd scripts
$ ./run.sh
```

기동 후 브라우저 접속 주소는 다음과 같습니다.

```text
http://<서버 IP>:8111
```

### 컨테이너 중지

중지 스크립트: `scripts/stop.sh`

```bash
$ cd scripts
$ ./stop.sh
```

## 클러스터 설치 절차

### 인벤토리 설정 화면

메뉴: `설정 파일 > 인스톨러 > 인벤토리 설정`

![installer_ui_inventory.png](/assets/images/v3/accordion-installer-ui/installer_ui_inventory.png)

아코디언 인스톨러는 Ansible 기반의 자동화 설치 도구입니다.
인벤토리 설정 화면에서는 실제 설치 대상 서버와 SSH 접속 정보를 입력합니다.
멤버 클러스터는 옵션이며, 호스트 클러스터만 설치할 경우 멤버 클러스터 영역은 비워두면 됩니다.

1. `마스터 수`를 `1대` 또는 `3대`로 선택합니다.
2. `마스터1`, 필요 시 `마스터2`, `마스터3`의 `Hostname`, `IP`, `NIC`를 입력합니다.
3. 인프라 노드가 있으면 `인프라 노드 추가`로 행을 늘려 입력합니다.
4. 워커 노드를 입력합니다.
5. 하단 `Ansible 접속 정보`에 `SSH 포트`, `사용자`, `사용자 그룹`, `SSH 패스워드`를 입력합니다.

설정 시 참고할 내용은 다음과 같습니다.

- 워커 노드는 최소 1대 이상 구성해야합니다.
- `SSH 패스워드`를 비우면 SSH Key 인증을 기본으로 사용합니다.
- `ansible_user`는 실제로 대상 노드에 접속 가능한 계정이어야 하며, sudo 권한을 갖고 있어야합니다.


### 클러스터 설정 화면

메뉴: `설정 파일 > 인스톨러 > 클러스터 설정`

![installer_ui_host_yml.png](/assets/images/v3/accordion-installer-ui/installer_ui_host_yml.png)

클러스터 설정 화면은 `아코디언 클러스터`의 설정을 입력하는 화면입니다.
클러스터 형태에 따라 `호스트 클러스터`와 `멤버 클러스터`로 나뉘며, 이미 기존에 K8s 클러스터가 존재하는 경우 `kubeconfig`를 통해 아코디언 클러스터로 조인 구성이 가능하도록 지원하고 있습니다.
섹션별로 필요한 값을 순서대로 정리하면 다음과 같습니다.

- 클러스터 설정: 클러스터의 이름, 런타임, 마스터 노드 구성 등을 설정합니다.
  - `클러스터 이름`
  - `마스터 노드 HA`
  - `마스터 격리 사용`
  - `컨테이너 런타임`

- LB 설정: 마스터 HA 구성 시 Load Balancer의 구성 정보를 입력합니다.
  - `LB 타입`
  - `LB IP`
  - `LB 포트`
  - `LB 도메인 사용`
  - `LB 도메인 주소`

- 스토리지 설정: 클러스터에서 사용할 데이터 저장 공간을 입력합니다.
  - `스토리지 타입`
  - `스토리지 경로`
  - `아코디언 스토리지 클래스명`
  - `사용자 스토리지클래스 사용 여부`

- 레지스트리 설정: K8s 클러스터에서 이미지 저장소로 사용할 레지스트리를 입력합니다.
  - `베이스 레지스트리 도메인`
  - `유저 레지스트리 도메인`
  - (옵션) 외부 레지스트리 사용 여부에 따른 주소, 포트, 계정 정보

- 네트워크 설정: K8s 클러스터에서 네트워킹을 구성할 정보를 입력합니다.
  - `CNI 선택`
  - `Podman CIDR`
  - `Pod CIDR`
  - `Service CIDR`
  - `kube-proxy 모드`

- 디렉터리 설정: 클러스터의 디렉토리 구성 정보를 입력합니다.
  - `기본 경로 사용`
  - (옵션) `사용자 경로 지정`
  - (옵션) `Kubernetes Addon 경로`
  - (옵션) `Docker RPM 경로`

- Add-on 패키지 설치 여부: 필요한 기능과 필요하지 않은 기능을 ON/OFF하여 필요한 기능만 설치될 수 있도록 구성합니다.
  - `GPU 모니터링`
  - `Istio`
  - `OTEL`
  - `Logging`
  - `APM`
  - `CICD`
  - `ANOM`

- HA 옵션: 아코디언 모듈을 HA 구성할 것인지에 대한 여부입니다.
  - `seaweed_ha`
  - `prometheus_ha`
  - `alertmanager_ha`
  - `elasticsearch_ha`
  - `logstash_ha`
  - `thanos_ha`
  - `tsdb_ha`
  - `clickhouse_ha`


### 클러스터 설치 실행

메뉴: `실행 > 작업 실행`

![installer_ui_runner.png](/assets/images/v3/accordion-installer-ui/installer_ui_runner.png)

클러스터 설정을 저장한 뒤에는 `실행` 화면으로 이동합니다.

1. 좌측 `구분`에서 최초 설치 시 `인스톨러`를 선택하고, 업그레이드를 수행하고자 하는 경우 `업그레이드`를 선택합니다.
2. 목록에서 설치/업그레이드 하고자 하는 슬롯 버튼을 클릭합니다.
3. `스크립트 실행` 버튼을 눌러 설치/업그레이드를 수행합니다.
4. 우측 라이브 콘솔에서 진행 상황을 확인할 수 있습니다. (브라우저를 종료해도 세션은 유지됩니다.)

## 보조 기능

### 최근 실행 이력

![installer_ui_exec_history.png](/assets/images/v3/accordion-installer-ui/installer_ui_exec_history.png)

`작업 실행` 화면의 `사용자 명령` 버튼을 클릭 후 도구 탭에서 최근 실행 이력을 바로 확인할 수 있습니다.

### 노드 정보 수집

![installer_ui_exec_get_facts.png](/assets/images/v3/accordion-installer-ui/installer_ui_exec_get_facts.png)

`accordion`, `upgrade`, `edge` 중 하나를 선택해 노드 정보 수집 결과를 조회할 수 있습니다.
설치할 노드 OS, 커널, IP 정보를 확인할 때 유용합니다.

### 방화벽 포트 체크

![installer_ui_exec_firewall.png](/assets/images/v3/accordion-installer-ui/installer_ui_exec_firewall.png)

포트 체크 기능은 인벤토리 기준으로 Source/Destination 포트를 지정해 통신 가능 여부를 확인합니다.

- 타겟 인벤토리 선택
- Source Role / Dest Role 선택
- 수동 IP 지정 가능
- 결과 JSON 복사 가능
- 실패 결과만 필터링 가능
