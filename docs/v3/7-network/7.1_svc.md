## 서비스(Service)

파드 집합에서 실행중인 애플리케이션을 네트워크 서비스로 노출하는 추상화 방법
쿠버네티스를 사용하면 익숙하지 않은 서비스 디스커버리 메커니즘을 사용하기 위해 애플리케이션을 수정할 필요가 없다. 쿠버네티스는 파드에게 고유한 IP 주소와 파드 집합에 대한 단일 DNS 명을 부여하고, 그것들 간에 로드-밸런스를 수행할 수 있다.

<br>

외부와 접하는 단일 엔드포인트 뒤에 있는 클러스터에서 실행되는 애플리케이션을 노출시키며, 이는 워크로드가 여러 백엔드로 나뉘어 있는 경우에도 가능하다.

## 서비스 생성

`+생성`을 클릭 후 내용을 입력하여 서비스를 생성할 수 있습니다. 서비스는 FORM/YAML로 입력할 수 있습니다.

일반 탭에서는 이름과 네임스페이스, 라벨, 어노테이션을 설정할 수 있습니다.

![service-003-0.png](/assets/images/v3/network/service/svc_create_normal.png)

<br>

설정 탭에서는 라벨 셀렉터와 서비스타입을 설정할 수 있습니다.

![service-003-1.png](/assets/images/v3/network/service/svc_create_detail.png)

**서비스(Service) 설정 항목**

| 설정 항목 | 설명 | 비고 |
|:---|:---| :---|
| **셀렉터 (Selector)** | 서비스를 연결할 파드들을 식별하기 위한 라벨(키, 값)을 설정합니다. | 서비스는 라벨로 파드를 식별합니다.|
| **서비스 타입** | 서비스의 노출 방식을 선택합니다. (ClusterIP, NodePort, LoadBalancer 등) | ClusterIP: 클러스터 내에서 접근할 수 있는 서비스입니다. <br> NodePort: 클러스터 외부에서 접속 할 수 있는 서비스입니다. <br> LoadBalancer: 클라우드 환경에서 NodePort를 로드밸런서로 접속 할 수 있게하는 서비스입니다. |
| **이름 (Name)** | 서비스 포트 설정의 식별자 이름을 입력합니다. | - |
| **프로토콜 (Protocol)** | 네트워크 통신에 사용할 프로토콜을 선택합니다. (TCP, UDP, SCTP) | - |
| **포트 (Port)** | 서비스가 외부(클러스터 내부 포함)로 노출할 포트 번호를 설정합니다. | - |
| **타겟 포트 (Target Port)** | 서비스가 트래픽을 전달할 실제 파드 내부의 포트 번호를 설정합니다. | 포트로 받아온 트래픽을 타겟 포트로 전달합니다. <br> 포트를 전달하기 위해서 파드에 포트(containerPort)가 오픈되어있어야합니다. |
