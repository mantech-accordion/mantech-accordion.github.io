## 인그레스(Ingress)

인그레스는 클러스터 외부에서 클러스터 내부 서비스로  HTTP(혹은 HTTPS) 네트워크 서비스를 가능하게 합니다. 인그레스는 쿠버네티스 API를 통해 정의한 규칙에 기반하여 트래픽을 다른 백엔드에 매핑할 수 있게 해줍니다.

> **주의:**
>
> 전제 조건:
> 인그레스 컨트롤러가 있어야 인그레스를 충족할 수 있습니다. 인그레스 리소스만 생성한다면 효과가 없습니다.
> 여러 인그레스 컨트롤러 중에서 선택할 수도 있습니다.
> 모든 인그레스 컨트롤러는 참조 사양에 맞아야 하지만, 실제로 다양한 인그레스 컨트롤러는 조금 다르게 작동할 수 있습니다.

## 인그레스 생성

`+생성`을 클릭 후 내용을 입력하여 인그레스를 생성할 수 있습니다. 인그레스는 FORM/YAML로 입력할 수 있습니다.

일반 탭에서는 이름과 네임스페이스, 라벨, 어노테이션을 설정할 수 있습니다.

![ingress-004.png](/assets/images/v3/network/ingress/ing_create_normal.png)

<br>

설정 탭에서는 인그레스 규칙을 설정할 수 있습니다.

![ingress-005.png](/assets/images/v3/network/ingress/ing_create_detail.png)

**인그레스(Ingress) 설정 항목**

| 구분 | 설정 항목 | 설명 |
|:---|:---|:---|
| **인그레스 설정** | **인그레스 클래스명** | 사용할 인그레스 컨트롤러를 지정합니다. |
| | **인그레스 전용 어노테이션** | 컨트롤러별 상세 기능(Rewrite, SSL Redirect 등)을 설정합니다. |
| **라우팅 규칙** | **도메인 주소 (Host)** | 서비스에 접근할 도메인을 입력합니다. |
| | **경로 (Path)** | 도메인 뒤의 세부 URL 경로를 설정합니다. (예: /api, /main) |
| | **경로 유형 (Path Type)** | 경로 매칭 방식입니다. (Prefix: 접두사 일치, Exact: 완전 일치) |
| | **백엔드 (Backend)** | 트래픽을 전달할 대상 서비스를 선택합니다. |
| | **포트 (Port)** | 대상 서비스의 포트 번호를 선택합니다. |
| **보안 (SSL)** | **SSL Redirect** | HTTP 접속 시 자동으로 HTTPS로 전환할지 여부를 설정합니다. |
| | **인증서 (TLS)** | HTTPS 보안 통신을 위한 도메인별 인증서를 설정합니다. |
| | **시크릿 (Secret)** | 인증서 파일(crt, key)이 저장된 시크릿 오브젝트를 연결합니다. |
