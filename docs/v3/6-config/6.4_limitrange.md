## 리밋레인지(LimitRanges)

리밋레인지는 네임스페이스의 각 적용 가능한 오브젝트 종류(예: 파드 또는 퍼시스턴트볼륨클레임(PersistentVolumeClaim))에 대해 지정할 수 있는 리소스 할당(제한 및 요청)을 제한하는 정책입니다.

> **주의:**
>
> 리밋레인지는 다음과 같은 제약 조건을 제공합니다.
>
>> 1. 네임스페이스에서 파드 또는 컨테이너별 최소 및 최대 컴퓨팅 리소스 사용량을 지정합니다.
>> 2. 네임스페이스에서 퍼시스턴트볼륨클레임별 최소 및 최대 스토리지 요청을 지정합니다.
>> 3. 네임스페이스에서 리소스에 대한 요청과 제한 사이의 비율을 지정합니다.
>> 4. 네임스페이스에서 컴퓨팅 리소스에 대한 기본 요청/제한을 설정하고 런타임에 있는 컨테이너에 자동으로 설정합니다.
{: .callout-warning }

<details>
<summary> 리밋레인지 예제 YAML</summary>
<div markdown="1">
{% highlight yaml %}
apiVersion: v1
kind: LimitRange
metadata:
  name: cpu-resource-constraint
spec:
  limits:
  - default: # 이 섹션에서는 기본 한도를 정의한다
      cpu: 500m
    defaultRequest: # 이 섹션에서는 기본 요청을 정의한다
      cpu: 500m
    max: # max와 min은 제한 범위를 정의한다
      cpu: "1"
    min:
      cpu: 100m

{% endhighlight %}
</div>
</details>

## 리밋레인지 생성

`+생성`을 클릭 후 내용을 입력하여 리밋레인지를 생성할 수 있습니다. 리밋레인지는 FORM/YAML로 입력할 수 있습니다.

일반 탭에서는 이름과 네임스페이스, 라벨, 어노테이션을 설정할 수 있습니다.

![limitrange-003-0.png](/assets/images/v3/config/limitrange/lrange_create_normal.png)

<br>

설정 탭에서는 리밋레인지의 상세 설정을 할 수 있습니다.

![limitrange-003-1.png](/assets/images/v3/config/limitrange/lrange_create_detail.png)

**리밋레인지(LimitRange) 설정 항목**

| 설정 항목 | 설명 |
|:---|:---|
| **최소 (min)** | 리소스 제한의 최소값을 설정합니다. |
| **최대 (max)** | 리소스 제한의 최대값을 설정합니다. |
| **기본요청 (defaultRequest)** | 파드 생성 시 리소스 요청(`requests`)이 생략된 경우 이 값으로 자동 설정됩니다. |
| **기본제한 (default)** | 파드 생성 시 리소스 제한(`limits`)이 생략된 경우 이 값으로 자동 설정됩니다. |

> **참고**
>
> - LimitRange는 파드 뿐만 아니라 PersistentVolumeClaim(PVC)에도 적용할 수 있습니다.
> - 리밋레인지 설정은 네임스페이스 단위로 적용됩니다.
{: .callout-warning }
