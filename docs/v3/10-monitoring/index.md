# 모니터링 및 로깅

모니터링은 클러스터별로 시스템, 서비스 관측, 컨트롤 플레인, 알림 로그, 이벤트 로그, 시스템 로그, 감사 로그, 대시보드를 제공합니다.
아코디언과 클러스터에서 실행 중인 서비스의 전반적인 상태를 확인하고, 장애 징후를 빠르게 탐지하기 위한 기능입니다.

| 메뉴 | 설명 |
|---|---|
| 시스템 | 클러스터 리소스 상태와 워크로드별 리소스 사용량을 확인합니다. |
| 서비스 관측 | OpenTelemetry 기반 서비스 맵, 트레이스, 로그를 분석합니다. |
| 컨트롤 플레인 | Kubernetes 제어 계층의 가용성, 오류율, 지연시간, 처리량을 확인합니다. |
| 알림 로그 | 임계치 알림, 이상탐지 알림, 장애예측 알림 로그를 확인합니다. |
| 이벤트 로그 | Kubernetes 이벤트 로그를 확인합니다. |
| 시스템 로그 | 시스템 네임스페이스의 컨테이너 로그를 확인합니다. |
| 감사 로그 | Kubernetes API 접근 감사 로그를 확인합니다. |
| 대시보드 | 모니터링 데이터를 시각화하기 위한 대시보드를 관리합니다. |
