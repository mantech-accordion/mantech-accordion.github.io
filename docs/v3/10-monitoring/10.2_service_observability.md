## 서비스 관측

서비스 관측은 OpenTelemetry 기반으로 서비스 간 호출 관계와 요청 흐름을 확인하는 메뉴입니다.
서비스 맵, 서비스, 트레이스, 로그를 통해 장애 지점과 원인을 단계적으로 분석할 수 있습니다.

---

## 서비스 맵

서비스 맵은 서비스 간 호출 관계를 그래프로 보여줍니다.
각 서비스에는 요청량, 오류율, P95 응답 시간이 표시됩니다.

![service-view_servicemap.png](/assets/images/v3/monitoring/service-view_servicemap.png){: width="1200" }

| 항목 | 설명 |
|---|---|
| 서비스 카드 | 서비스명, 네임스페이스, 애플리케이션, 요청량, 오류율, 응답 시간을 표시합니다. |
| 연결선 | 서비스 간 호출 방향과 프로토콜을 표시합니다. |
| 필터 | 서비스, 네임스페이스, 애플리케이션, 프로토콜 기준으로 표시 대상을 제한합니다. |
| 성능 지표 | 요청 수, 에러율, 응답 시간이 높은 상위 서비스를 차트로 보여줍니다. |

---

## 서비스

서비스 탭은 서비스별 요청량, 오류율, 응답 시간을 테이블로 비교하는 화면입니다.
문제가 있는 서비스를 빠르게 찾을 때 사용합니다.

![service-view_service.png](/assets/images/v3/monitoring/service-view_service.png){: width="1200" }

| 컬럼 | 설명 |
|---|---|
| 서비스 | 서비스 이름 |
| 네임스페이스 | 서비스가 속한 네임스페이스 |
| 애플리케이션 | 서비스가 속한 애플리케이션명 |
| 요청 수 | 요청량과 추이 |
| 에러율 | 오류 비율 |
| 응답 시간 | P95 응답 시간 |

---

## 트레이스

트레이스 탭은 요청 1건이 여러 서비스와 작업을 거치는 흐름을 확인하는 화면입니다.
응답 지연, SQL 시간, 상태 코드, 예외 정보를 기준으로 문제 요청을 분석합니다.

![service-view_trace.png](/assets/images/v3/monitoring/service-view_trace.png){: width="1200" }

| 항목 | 설명 |
|---|---|
| 트레이스 분포 차트 | 시간과 응답 시간 기준으로 요청 분포를 보여줍니다. |
| 필터 | 서비스명, 스팬명, 상태, 응답 시간, 태그 기준으로 조회합니다. |
| 트레이스 목록 | 시작 시간, 서비스, 트레이스 이름, 응답 시간, SQL 시간, 상태 코드를 표시합니다. |
| 상세 정보 | 선택한 트레이스의 워터폴, 스팬 속성, 이벤트, 로그를 확인합니다. |

---

## 로그

로그 탭은 서비스에서 발생한 로그를 조회하는 화면입니다.
트레이스나 메트릭에서 확인한 이상 현상의 실제 로그 메시지를 확인할 때 사용합니다.

![service-view_log.png](/assets/images/v3/monitoring/service-view_log.png){: width="1200" }

| 항목 | 설명 |
|---|---|
| 라벨 필터 | 서비스명, 네임스페이스, 파드명 등 로그 속성으로 필터링합니다. |
| 본문 포함 | 로그 메시지에 포함된 키워드로 검색합니다. |
| 로그 카운트 차트 | 시간대별 로그 발생 건수를 보여줍니다. |
| 로그 테이블 | 로그 발생 시간과 메시지 본문을 표시합니다. |

---

## 운영 참고

서비스 관측은 트레이스, 메트릭, 로그 데이터를 저장하므로 대상 애플리케이션이 많아질수록 저장소 사용량과 쿼리 부하가 증가할 수 있습니다.
운영 환경에서는 수집량, 보관 주기, 샘플링 설정을 함께 관리합니다.
