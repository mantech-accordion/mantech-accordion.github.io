## 감사 로그

감사 로그는 Kubernetes API 서버로 들어오는 리소스 접근 요청을 조회하는 메뉴입니다.
누가, 언제, 어떤 리소스에, 어떤 동작을 수행했는지 확인할 수 있어 보안 감사와 이상 접근 탐지에 활용합니다.

> 사진 필요: 감사 로그 기본 화면  
> 캡처 범위: 상단 필터, 우측 조회/새로고침 영역, 차트, 하단 테이블이 함께 보이는 화면

---

## 필터 조건

| 항목 | 설명 |
|---|---|
| 네임스페이스 | 접근 대상 리소스의 네임스페이스 |
| 사용자명 | 요청을 수행한 사용자 또는 ServiceAccount |
| Verb | get, list, create, update, delete 등 API 동사 |
| 응답상태 | API 요청의 HTTP 응답 코드 |

> 사진 필요: 감사 로그 필터 선택 화면  
> 캡처 범위: 필터 드롭다운이 열린 상태 또는 조건이 추가된 상태

---

## 화면 구성

| 영역 | 설명 |
|---|---|
| 검색 결과 | 현재 조건에 해당하는 감사 로그 건수를 표시합니다. |
| 로그 카운트 차트 | 시간대별 감사 로그 발생 건수를 표시합니다. |
| 감사 로그 테이블 | 개별 API 접근 로그를 목록으로 표시합니다. |
| 상세 정보 | 선택한 감사 이벤트의 JSON 상세 정보를 표시합니다. |

---

## 테이블 항목

| 항목 | 설명 |
|---|---|
| 번호 | 로그 순번 |
| 시간 | 감사 이벤트 발생 일시 |
| 네임스페이스 | 접근 대상 리소스의 네임스페이스 |
| Verb | API 동사 |
| 응답상태 | HTTP 응답 코드 |
| 사용자IP | 요청이 발생한 클라이언트 IP |
| 사용자명 | 요청을 수행한 사용자 |
| 요청URI | API 요청 경로 |

---

## 상세 정보

테이블에서 로그를 선택하면 감사 이벤트의 전체 JSON 정보를 확인할 수 있습니다.
요청 사용자, 대상 리소스, 요청 URI, 응답 상태, RBAC 인가 결과를 확인할 수 있습니다.

> 사진 필요: 감사 로그 상세 화면  
> 캡처 범위: 테이블 행을 선택한 뒤 우측 상세 패널이 열린 상태
